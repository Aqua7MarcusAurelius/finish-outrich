# 9.5 — Починка tsconfig composite-references + два type-error

**Дата:** 2026-04-26
**Этап:** 9.5 (мелкий cleanup)

## Что сделано

IDE подсвечивала `modules/ui_web/tsconfig.json` оранжевым — TypeScript
ругался что в `references` лежит проект без `composite: true`. Починил
конфиг, побочно вылезли два настоящих type-error которые Vite в dev-режиме
тихо пропускал. Поправил их тоже + добавил `*.tsbuildinfo` и сгенерённые
`vite.config.{js,d.ts}` в `.gitignore`.

## Файлы

- `modules/ui_web/tsconfig.node.json` — добавлено `"composite": true`,
  убрано `"noEmit": true` (нельзя в composite-проекте по требованию TS).
- `modules/ui_web/src/components/events/EventFilters.tsx` — заменил
  устаревшее значение `"telegram"` (нет в `BusModule`) на актуальный
  список модулей (wrapper, history, history_sync, transcription,
  description, auth, worker_manager, autochat, api, cleaner).
- `modules/ui_web/src/components/events/EventRow.tsx` — типизация
  `onQuickFilter` сделана через `Partial<EventFilters>` вместо
  inline-литерала. Раньше типы расходились с `EventLogPage::applyQuickFilter`.
- `.gitignore` — добавлены `node_modules/`, `*.tsbuildinfo`,
  `modules/ui_web/dist/`, `vite.config.{d.ts,js}`.

## Решения

### `composite: true` обязательно для referenced-проекта
Без него `tsc -b` (build mode по project references, как в `npm run build`)
кидает TS6306. С `composite` нельзя `noEmit: true` — TS шлёт TS6310. Так
что замена не косметическая: либо composite + emit, либо ничего.

В tsconfig.node.json теперь `"composite": true` без `noEmit` — TS будет
эмиттить декларации (`vite.config.d.ts`) и `.tsbuildinfo` рядом. Эти файлы
бесполезны для прода (Vite читает `.ts` напрямую), коммитить не надо —
ушли в `.gitignore`.

### Старые type-errors вылезли потому что Vite их игнорил
В dev-режиме Vite использует esbuild для трансформации, esbuild не
проверяет типы — только синтаксис. Поэтому `"telegram"` в литерале и
type-mismatch в onQuickFilter жили незамеченными. `tsc -b` теперь
реально прогоняет проверку типов и эти проблемы вышли. Хорошо что вышли
— "telegram" в фильтре никогда бы ничего не отфильтровал, был мёртвый
выбор.

### `BusModule` строгий, а EventRow.onQuickFilter был с `string`
Расхождение типов: `EventLogPage` передавал колбэк `(patch: Partial<EventFilters>) => void`
где `module: BusModule`, а `EventRow` принимал `(patch: { module?: string }) => void`.
TS видел что `string` не совместим с `BusModule | undefined`. Унифицировал
через общий `Partial<EventFilters>` — теперь источник правды в одном месте.

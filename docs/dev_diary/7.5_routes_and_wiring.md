## Ключевые решения

### 4 endpoint'а, один POST /start как "тяжёлый"

```
POST   /autochat/start              — sync-флоу: resolve → LLM → send → active
GET    /autochat/sessions           — list + фильтры account_id/status
GET    /autochat/sessions/{id}      — один
POST   /autochat/sessions/{id}/stop — graceful stop (идемпотентен)
```

Остальное (pause/resume, SSE-стрим событий сессии) отложено.
Pause — не нужно на MVP, логичнее просто stop и создать новую.
SSE — можно подписаться на глобальный `/events/stream` с фильтром
по `module=autochat` и `account_id=X`.

### Ошибки маппятся через общий `_err`

Симметрично `modules/auth/routes.py::_err`: берём `AutoChatError`,
возвращаем `JSONResponse(status_code=e.status_code, content={"error":
{"code": e.code, "message": e.message}})`. Единственное место
маппинга — не дублируем формат по endpoint'ам.

### Wiring в lifespan — после WorkerManager, первым на shutdown

```python
# Startup:
app.state.worker_manager = WorkerManager()
autochat_service = AutoChatService(
    get_wrapper=app.state.worker_manager.get_wrapper,
)
autochat_task = asyncio.create_task(autochat_service.run())

# Shutdown (порядок критичен):
await autochat_service.stop()    # ← первым
await app.state.worker_manager.shutdown()
...
```

**Почему AutoChat стопается первым:** если сначала остановить
WorkerManager, воркеры закроют Telegram-соединения, и оставшиеся
сегменты в `send_queue` не смогут отправиться. Сначала даём
AutoChat'у дописать текущие сегменты (`session.stop()` дожидается
sender'а с таймаутом 5 сек), потом гасим воркеры.

Таймаут 5 сек выбран так: типовой сегмент ~2-5 сек печати + отправка
+ пауза. Одного сегмента хватит дописать; следующие если остались —
потеряются. Это ок: даже без AutoChat отключение приложения в
середине отправки это не катастрофа — собеседник увидит то что
успели.

### Передача WorkerManager в AutoChatService через Callable

```python
AutoChatService(get_wrapper=app.state.worker_manager.get_wrapper)
```

а не `AutoChatService(worker_manager=...)`. **Почему:** сервису
нужен только один метод — `get_wrapper(account_id) -> TelegramWrapper
| None`. Передавать целый WorkerManager — это утечка абстракции
(AutoChat не должен знать как запускаются воркеры).

Один контракт, одна зависимость — проще тестировать, проще
заменять.

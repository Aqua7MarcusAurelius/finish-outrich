# 8.5 — Диалоги: колонка сообщений прижата к левому краю

**Дата:** 2026-04-24
**Этап:** 8.5
**Коммит:** `aabe8da`

## Что сделано

Убрал `mx-auto` у шапки диалога и у контейнера ленты. Колонка шириной 780px теперь примыкает к левому краю правой панели; всё пустое пространство собрано справа — там будет виджет.

## Файлы

- [modules/ui_web/src/pages/DialogsPage.tsx:99-126](../../modules/ui_web/src/pages/DialogsPage.tsx#L99-L126) — удалено `mx-auto` в двух местах (header-контейнер и message-контейнер), обновлён комментарий.

## Почему так

В 8.4 колонка стояла по центру с двумя равными полями (Telegram Desktop-style). Пользователь попросил прижать влево — будущий виджет (профиль собеседника или related event log) хочет жить справа цельным блоком в ~600px, а не двумя кусками по 300px слева/справа. Цельное поле удобнее и для широких таблиц, и для лейаута с sticky-хедером.

Итоговый лейаут правой панели после фикса:

```
┌─── 780px (шапка + лента) ───┬──── остаток (будущий виджет) ────┐
│ Test account · @vians777     │                                  │
│ ─── bubble ───────           │                                  │
│          ─── bubble ──       │                                  │
└──────────────────────────────┴──────────────────────────────────┘
```

## Открытые вопросы

- **Виджет справа** — не делаю сейчас, пользователь захотел сначала решить ориентацию чата. План виджета см. 8.4.
- **Overflow на узких экранах.** При ширине правой панели <780px колонка будет сжиматься (благодаря `max-w-`). Проверял только FullHD.

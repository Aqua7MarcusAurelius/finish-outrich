## Почему так, а не иначе

### Три маршрута по типу media

```
photo, sticker           → describe_images([raw])         (одна картинка)
gif, video, video_note   → extract_frames → describe_images(frames)
document                 → describe_document(raw)
```

Множества вынесены в константы `STATIC_IMAGE_TYPES`, `FRAME_TYPES`,
`DOCUMENT_TYPES` — читаемо и одинаково используется в фильтре входа и
в dispatcher.

### Зеркало transcription, но с тремя отличиями

1. **Флаги на входе** — `has_image or has_video or has_document` (у
   транскрипции было `has_audio or has_video`). Video_note имеет и
   `has_audio` и `has_video` → попадает к обоим модулям, каждый делает
   своё поле (`transcription` и `description`) — это by design.
2. **Политика ретраев вынесена в `_retry_policy`** — принимает callable,
   потому что для изображений и документов вызываемая функция разная.
   В транскрипции была одна функция (`transcribe_audio`), там вынос не
   нужен был. Логика самой политики та же: `1 + retries` попыток на
   ошибку, один повтор на пустое, пустое на финале — это валидный
   `done`.
3. **Настройки из БД — две** (`description.retries` и
   `description.frames_count`), поэтому добавлен маленький хелпер
   `_get_int_setting(key, default)`. У транскрипции настройка одна, там
   хелпер не нужен.

### Filename для документов — из MIME

OpenRouter требует поле `filename` в `type: "file"`. Реальное имя файла
в payload `message.saved` не приходит (в `saved_media` хранятся только
`media_id/type/storage_key/mime_type/duration`). Подтягивать из БД на
каждый кейс — лишний запрос. Вместо этого генерируем из MIME:
`application/pdf → document.pdf`, `image/png → document.png`, и т.п.
Gemini смотрит на `mime_type` из самой data-URL, поле `filename` —
только метка в логах провайдера.

### image_format для статичных картинок

Фото от Telegram — всегда JPEG, стикеры — WebP (или TGS, но TGS мы в
MinIO не кладём). Парсим MIME: `image/webp → webp`, `image/png → png`,
всё остальное → jpeg. Неправильный формат в data-URL → модель откажется
читать.

### Ошибки ffmpeg — два разных выхода

- `NoFramesError` (нет видеодорожки, длительность 0) → `done` с пустым
  текстом, как в транскрипции `NoAudioError`. Не ошибка — просто
  «описывать нечего».
- `FfmpegError` → `failed` с `ffmpeg: {msg}` в поле error.

Это соблюдает принцип: **пустой результат ≠ ошибка**.

## Открытые вопросы

- **Документы, не поддерживаемые Gemini нативно** (xlsx, docx). OpenRouter
  при `engine=native` попытается отдать нативно → Gemini откажет → в
  ответе будет текст ошибки. Попадёт в `failed` с сообщением. При
  необходимости — переключиться на `engine=mistral-ocr` или
  `cloudflare-ai` для документов, но это отдельное решение.
- **Размер входа.** Не ограничивается здесь. HTTP-таймаут 180с; в
  отправку падают base64-строки. Если встретим TB-овый PDF — это
  внешняя проблема. Сам HTTP-клиент поднимет сетевую ошибку.
- **Порядок запуска** сервиса в lifespan — будет в 6.4.

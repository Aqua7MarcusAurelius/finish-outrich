# Дневник разработки

Журнал решений и изменений по ходу работы. В отличие от `docs/STAGE_N_REPORT.md`
(итоговых отчётов по этапу) — здесь ведётся **пошагово**, по мере реализации.
Каждая запись — отдельный файл с номером подэтапа.

Цель: если через месяц спросят «почему тут именно gemini-2.5-flash, а не gpt-4o» —
ответ должен найтись тут же, без археологии по git-логу.

## Формат записи

- **Имя файла:** `{этап}.{подэтап}_{короткое_имя}.md` — напр. `6.1_openrouter_client.md`
- **Секции:**
  - *Что сделано* — в двух строках
  - *Файлы* — конкретные пути
  - *Почему так, а не иначе* — ключевые развилки и выбор
  - *Открытые вопросы* — если есть

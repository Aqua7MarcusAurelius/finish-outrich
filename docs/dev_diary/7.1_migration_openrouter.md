## Ключевые решения

### Partial unique index, не просто UNIQUE

```sql
CREATE UNIQUE INDEX autochat_sessions_active_pair_uniq
ON autochat_sessions (account_id, telegram_user_id)
WHERE status IN ('active', 'paused', 'starting');
```

**Почему:** завершённые (`failed`, `stopped`) сессии не должны блокировать
повторный запуск автодиалога с тем же собеседником. Если отвалились —
можно стартовать новую с другим промтом. Partial unique держит
инвариант "две активных одновременно нельзя", но не мешает истории.

### `chat_completion` как отдельный универсальный метод

Остальные публичные функции `core/openrouter.py` (`transcribe_audio`,
`describe_images`, `describe_document`) — узкоспециализированные
под конкретное content-поле (input_audio / image_url / file). Для
AutoChat контент — только текстовые messages, никаких приложений.
Общий `chat_completion(messages, model, temperature, max_tokens)` —
тонкая обёртка над тем же приватным `_post()`. Не дублирует код,
но семантически отделён.

### Константа модели отдельная

Не переиспользуем `OPENROUTER_MODEL_DESCRIPTION` или `_TRANSCRIPTION` —
это разные задачи с разными моделями (vision / audio / reasoning).
Новая `OPENROUTER_MODEL_AUTOCHAT` с дефолтом `anthropic/claude-opus-4.7`
позволит легко переключать без касания кода.

### Настройки в БД, не в .env

Все тайминги (enter 15/60/120, idle 180, reply 30, retries 2,
typing 40 мс/символ) живут в `settings`, т.е. в `autochat_sessions` их
не копируем. Это позволит тюнить поведение без рестарта — у нас уже
есть этот паттерн для `transcription.retries`, `description.retries`,
`cleaner.*` и прочих.

### 10 новых событий + 2 модуля

`Module.AUTOCHAT` очевиден. `Module.BUS` добавил заранее —
`record_failure()` в `core/bus.py` (dead-letter из 6.9) публикует
`system.error` с `module="bus"`, но эта строка там была захардкожена.
Позволит не копить inline-строки.

Типы событий:
- 9 × `autochat.*` — жизненный цикл сессии (`started`, `initial_sent`,
  `entered_chat`, `left_chat`, `generation_requested`, `generation_done`,
  `segment_sent`, `session_stopped`, `session_error`).
- `dialog.typing_observed` — публикует враппер, слушает AutoChat
  (остальные модули его игнорируют).

Шаблоны в `event_messages.py` сделаны так, чтобы в UI-логе было
сразу понятно что происходит — "автодиалог #42 с @durov: вошли в
чат (задержка 60с)", а не "autochat.entered_chat ok".

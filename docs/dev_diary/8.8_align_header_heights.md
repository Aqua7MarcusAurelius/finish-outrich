# 8.8 — Диалоги: выровнял высоту шапки поиска и шапки диалога

**Дата:** 2026-04-24
**Этап:** 8.8
**Коммит:** `ae663e1`

## Что сделано

Шапка над списком диалогов (с input'ом поиска) и шапка секции сообщений (имя собеседника + @username + dialog #N) встали на одну высоту — нижний бордер теперь одна сквозная линия от края до края, без ступеньки.

## Файлы

- [modules/ui_web/src/pages/DialogsPage.tsx:87-103](../../modules/ui_web/src/pages/DialogsPage.tsx#L87-L103)
  — добавил `flex h-12 shrink-0 items-center px-2` в обе шапки, убрал `p-2` и отдельный div-wrapper.

## Проблема

Левая (поиск):
```tsx
<div className="border-b border-border p-2">   // p-2 = 8px сверху/снизу
  <Input className="h-8 ..." />                 // Input 32px
</div>
// итого ~48px + 1px border
```

Правая (dialog header):
```tsx
<header className="border-b border-border">
  <div className="flex items-center gap-3 p-2">  // p-2 + контент ~14-20px
    ...две строки текста...
  </div>
</header>
// итого ~36px + 1px border
```

Разница ~12px. Визуально — ступенька на границе aside/section, нижняя линия шапки идёт не в одну полосу.

## Решение

Один tailwind-класс `h-12` на обе шапки + общая структура `flex items-center px-2`:

```tsx
{/* search bar */}
<div className="flex h-12 shrink-0 items-center border-b border-border px-2">
  <Input placeholder="Поиск по диалогам…" ... />
</div>

{/* dialog header */}
<header className="flex h-12 shrink-0 items-center border-b border-border px-2">
  ...
</header>
```

`h-12` = 48px ровно, `shrink-0` защищает от flex-сжатия когда контент внизу растёт.

### Почему именно h-12, не auto

`h-auto` требует точного подбора padding'а чтобы совпали две разные внутренние структуры (Input vs текстовые блоки). Хрупко — любое изменение Input.tsx ломает выравнивание. Жёсткая высота `h-12` дешевле читать и правда перекликается с AppShell.tsx (там шапка `h-11` / 44px). Если когда-то решим унифицировать все шапки — одну цифру поменять.

### px-2 вместо p-2

Нужен только горизонтальный padding, вертикальный покрывает `items-center` внутри фиксированной высоты.

## Проверка

Открыть http://localhost:3000/#/dialogs/1/95: красной стрелки на скриншоте больше не должно быть — нижняя граница шапки идёт одной сквозной линией.

## Открытые вопросы

- Шапка приложения наверху (`AppShell.tsx`) — `h-11` (44px). Несоответствие с внутренними шапками `h-12` — в глаз не бросается, но если унифицировать, выбрать одно число и применить к обеим. Пока не трогаю.

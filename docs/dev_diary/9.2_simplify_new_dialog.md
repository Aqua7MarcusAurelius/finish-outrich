# 9.2 — Упрощение "+ Новый авто-диалог" до одного поля @username

**Дата:** 2026-04-26
**Этап:** 9.2 (после 9.0/9.1 — структурированный per-worker промт + превью)

## Что сделано

После того как поведение бота полностью переехало в per-worker конфиг
(account_prompts, страница `/workers/{id}/prompt`), per-session inputs
в окне создания нового авто-диалога стали лишним дублированием. Окно
ужалось до одного поля `@username` — всё остальное берётся из
сохранённого промта воркера.

## Файлы

### Backend
- `modules/autochat/routes.py` — `StartIn` оставил только `account_id`
  и `username`, выкинул `system_prompt` и `initial_prompt`. Вызов
  сервиса упрощён.
- `modules/autochat/service.py::create_session` — параметры
  `system_prompt` и `initial_prompt` удалены. INSERT в
  `autochat_sessions` пишет пустые строки в одноимённые NOT NULL
  колонки (схему не меняем, не нужно — это стоимость удержания
  обратной совместимости с архивом старых сессий).
- `modules/autochat/generation.py::build_initial_messages` — параметры
  `system_prompt` и `initial_prompt` удалены. Устаревшие плейсхолдеры
  `{user_system_prompt}` / `{user_initial_prompt}` (если оператор оставил
  их в шаблоне) тихо подставляются пустой строкой — не падаем и не
  печатаем литерально. `{current_time}` теперь в формате
  `DD.MM.YYYY HH:MM:SS` (унификация с reply-промтом и историей).

### Frontend
- `modules/ui_web/src/components/dialogs/NewDialogDialog.tsx` —
  переписан: одно поле `@username`, без двух textarea. Добавлен Enter
  для submit и подпись-напоминание "берётся из промта воркера".
- `modules/ui_web/src/lib/api.ts::autochatStart` — body упрощён до
  `{account_id, username}`.

## Решения и почему

### Колонки `system_prompt`/`initial_prompt` оставлены в `autochat_sessions`
Они NOT NULL. Дроп = миграция = риск на проде. Записываем пустые
строки в новые сессии. Старые остаются нетронутыми. В `_row_to_dict()`
поля по-прежнему отдаются — фронт их сейчас не использует, но и не
ломается.

Если когда-нибудь захотим почистить — отдельная миграция, отдельный юнит.

### Устаревшие плейсхолдеры → пустая строка, не warning
Если оператор скопировал старый initial_system из файла-фолбэка, там
есть `{user_system_prompt}` и `{user_initial_prompt}`. Без подстановки
они уйдут в LLM литералом — выглядит как баг шаблона. Тихая подстановка
пустой строкой = безопасное поведение для legacy. Warning в логе
было бы шумом без ценности (оператор увидит как промт работает в
превью на странице).

### Per-worker `initial_system` стал self-contained
Теперь оператор пишет в это поле всё сам: персонаж, сценарий, формат,
запреты. Никаких placeholder-инструкций "сюда подставится X".
Преимущество: один источник правды для initial-генерации, никаких
скрытых составных частей. Минус: оператор должен помнить что это
самостоятельный промт — но он же сам пишет, проблема не возникает.

### Enter в поле @username = submit
Лишнее ускорение для частой операции. Один клик / Enter — и пошёл
запрос. Без `Shift+Enter` поведение поля Input по умолчанию.

## Гейты при /autochat/start (текущее состояние)

1. Не запущен воркер аккаунта → 409 `WORKER_NOT_RUNNING`
2. Пустой `account_prompts.initial_system` → 409 `PROMPT_NOT_CONFIGURED`
3. `@username` не резолвится → 404 `USERNAME_NOT_FOUND`
4. Уже есть активная сессия на пару (account_id, username) → 409
   `SESSION_ALREADY_ACTIVE`
5. LLM вернул пустое первое сообщение → 502 `GENERATION_FAILED`
6. wrapper.send_message упал → 410 / 409 в зависимости от причины

## Что НЕ сделано

- Дроп колонок `system_prompt`/`initial_prompt` из `autochat_sessions` —
  пока не нужно, миграция отдельным юнитом если захотим почистить.
- Удаление поля initial_system из textarea на странице промта — наоборот,
  это поле теперь критично, гейт его проверяет.

## Sanity-check

```
$ curl -X POST /autochat/start -d '{"account_id":1,"username":"durov"}'
{"error":{"code":"WORKER_NOT_RUNNING","message":"Воркер аккаунта не запущен"}}

# С лишними полями (legacy клиенты) — Pydantic игнорирует, не падает
$ curl -X POST /autochat/start -d '{"account_id":1,"username":"durov","initial_prompt":"old","system_prompt":"old"}'
{"error":{"code":"WORKER_NOT_RUNNING","message":"Воркер аккаунта не запущен"}}
```

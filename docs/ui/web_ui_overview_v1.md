# Веб-интерфейс системы

> Панель наблюдения и управления для Telegram Automation Framework.  
> Построена поверх существующего API (FastAPI) — внутренняя логика не меняется.

---

## Область применения

Веб-интерфейс — это тонкий слой поверх API. Он не хранит собственного состояния, не содержит бизнес-логики и не ходит в базу напрямую. Всё что ему нужно — получает через API, всё что хочет изменить — отправляет командой в API.

Если веб-интерфейс временно недоступен — система продолжает работать. Если API временно недоступен — интерфейс показывает ошибку и не валится.

## Контракт с бэкендом

Интерфейс не ходит в PostgreSQL, Redis и MinIO напрямую. Все данные — только через API.

```
[Веб-интерфейс]
      │
      ▼
[API — FastAPI]
      │
      ├──▶ PostgreSQL (аккаунты, диалоги, сообщения, архив событий)
      ├──▶ Redis      (статусы воркеров, живой поток событий)
      └──▶ MinIO      (медиафайлы — ссылки на превью через API-проксю)
```

Какие именно ручки нужны — в `web_ui_api_contract_v1.md`.

## Общие принципы дизайна

Интерфейс — инструмент, а не витрина. Он должен быть плотным, читаемым и быстрым.

1. **Плоский** — никаких теней, градиентов, свечений. Белые поверхности, тонкие границы 0.5px.
2. **Компактный** — максимум информации в поле зрения без скролла. Декоративные отступы — минимальные.
3. **Цвет несёт смысл** — не декорация. Зелёный = success, красный = error, жёлтый = in_progress. У каждого модуля на шине свой постоянный цвет — он одинаковый на всех экранах.
4. **Моноширинный шрифт для технических полей** — id, storage_key, prefix событий, номера телефонов. Для всего остального — обычный.
5. **Тёмная тема** — поддерживается из коробки, не опционально.

## Цветовая разметка модулей

Модули шины событий всегда показываются одним и тем же цветом на любой странице. Это позволяет распознавать поток работы с одного взгляда.

| Модуль | Цвет | Где видим |
|---|---|---|
| `telegram` (враппер) | фиолетовый | event log, события получения сообщений |
| `history` | зелёно-бирюзовый | event log, события записи в БД |
| `transcription` | синий | event log, блок транскрипции в сообщении |
| `description` | розовый | event log, блок описания медиа в сообщении |
| `auth` | янтарный | event log, события авторизации и протухания сессий |
| `worker_manager` | нейтрально-серый | event log, статусы воркеров |

---

## Цветовая разметка статусов

Применяется к статусам событий на шине и к статусам воркеров.

| Статус | Цвет | Значение |
|---|---|---|
| `success` / `running` | зелёный | всё хорошо |
| `error` / `crashed` | красный | требует внимания |
| `in_progress` | жёлтый | работа идёт |
| `stopped` | серый | остановлен намеренно |

## Навигация между страницами

Страницы связаны — клик по сущности ведёт к её детальному просмотру.

```
[Диалоги]
   │
   ├── клик по аккаунту → фильтр в event log по этому аккаунту
   ├── клик по диалогу → сообщения этого диалога
   └── клик по сообщению с медиа → связанные события (transcription/description)

[Event log]
   │
   ├── клик по событию → полный payload + цепочка parent_id
   ├── клик по account_N → фильтр по аккаунту (оставаясь на event log)
   └── клик по message_id в событии → переход в диалоги к этому сообщению
```

## Что не делаем в v1

- **Отправку сообщений из интерфейса.** Только просмотр.
- **Редактирование собственных сообщений.** Только просмотр.
- **Загрузку новых медиа.** Только просмотр того что пришло.
- **Графики и дашборды.** Метрики — только числами в карточках сверху event log.
- **Настройки модулей.** Пока — только через API или конфиг.
- **Роли и права.** Один пользователь = один интерфейс. Аутентификация UI — на уровне инфраструктуры (Basic auth / reverse proxy).

Эти пункты добавляем отдельными модулями, когда понадобятся.

## Принципы которые мы зафиксировали

1. Веб-интерфейс — тонкий слой поверх API, без собственного состояния
2. Интерфейс не ходит в БД/Redis/MinIO напрямую — только через API
3. Падение интерфейса не влияет на систему, падение API показывается как ошибка
4. Цвет у модуля и статуса — постоянный на всех страницах
5. Интерфейс плоский, плотный, с поддержкой тёмной темы по умолчанию
6. Клик по сущности ведёт к её детальному просмотру или фильтру
7. В v1 — только просмотр и базовое управление воркерами

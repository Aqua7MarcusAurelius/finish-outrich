# Страница «Диалоги»

> Просмотр истории общения по всем нашим аккаунтам.  
> Только чтение — ни отправки, ни редактирования, ни удаления.

---

## Что показывает

Всё что записал модуль истории: список наших аккаунтов со статусами воркеров, список диалогов каждого аккаунта, и сообщения внутри диалога — с медиа, транскрипциями, описаниями, пересылками, правками и реакциями.

---

## Структура экрана

Три зоны сверху вниз:

```
┌────────────────────────────────────────────────────────────┐
│  [Карточка аккаунта 1] [Карточка аккаунта 2] [Карточка 3]  │  ← шапка
├───────────────────────┬────────────────────────────────────┤
│  поиск                │  собеседник, username, dialog #N   │
│  ─────────────────    │  ─────────────────────────────     │
│  [Диалог 1]           │  [входящее сообщение]              │
│  [Диалог 2]  ← выбран │  [исходящее сообщение]             │
│  [Диалог 3]           │  [входящее с медиа + описание]     │
│  [Диалог 4]           │  [входящее с голосовым + текст]    │
│  ...                  │  [удалённое собеседником]          │
│                       │  [редактированное, с reply]        │
│                       │  [с реакцией]                      │
├───────────────────────┴────────────────────────────────────┤
│  last events: message.saved · transcription.done · ...     │  ← футер
└────────────────────────────────────────────────────────────┘
```

**Шапка — карточки аккаунтов.** Горизонтальный ряд. Каждая карточка — один наш аккаунт. Показывает `name`, `phone`, статус воркера из Redis, количество диалогов и кнопку действия (Start / Stop / Reauth в зависимости от статуса). Клик по карточке выбирает аккаунт — диалоги в левой колонке обновляются.

**Левая колонка — диалоги.** Фиксированная ширина ~230px. Сверху — поле поиска. Ниже — список диалогов выбранного аккаунта. Каждая строка: аватар с инициалами, имя, @username, превью последнего сообщения, время. Отсортированы по времени последнего сообщения, новые сверху.

**Правая колонка — сообщения.** Остальная ширина. Сверху — шапка с именем собеседника, его @username, телефоном и внутренним `dialog #N`. Ниже — лента сообщений с прокруткой.

**Футер — бегущая строка событий.** Последние типы событий на шине по выбранному аккаунту. Клик — переход в event log с уже выставленным фильтром.

## Данные страницы

Всё что рендерится — из таблиц модуля истории.

| Элемент экрана | Откуда берём |
|---|---|
| Карточки аккаунтов | `accounts` + статус из Redis |
| Количество диалогов на карточке | `count(dialogs where account_id = N)` |
| Строка в списке диалогов | `dialogs` + последнее сообщение через `(dialog_id, date DESC)` |
| Превью последнего сообщения | `messages.text` или метка типа медиа |
| Заголовок в правой колонке | `dialogs` (username, phone, имя) |
| Сообщение | `messages` |
| Медиа в сообщении | `media` (все записи с `message_id`) |
| Транскрипция под медиа | `media.transcription` |
| Описание под медиа | `media.description` |
| Плашка «reply» | `messages.reply_to_message_id` → `messages.text` |
| Блок «переслано от» | `messages.forward_from_*` |
| Пометка «изменено» | `messages.edited_at` не null |
| Приглушённое сообщение «удалено» | `messages.deleted_at` не null |
| Реакции | `reactions` |

## Особые случаи в отрисовке сообщения

**Исходящее / входящее.** Управляется флагом `is_outgoing`. Исходящие — справа, синяя заливка. Входящие — слева, нейтральная серая. Никаких «аватаров» слева/справа у каждого пузыря — собеседник и аватар один на весь диалог, он в шапке.

**Медиа-тег.** Над содержимым сообщения — короткая моноширинная плашка: тип, размеры, размер файла, длительность. Пример: `voice · 14 сек`, `photo · 1080×1920 · 284 КБ`, `document · requirements_v3.pdf · 1.2 МБ`. Берём из `media`.

**Транскрипция.** Отдельный блок под медиа, со светло-серой подложкой и лейблом `TRANSCRIPTION · Whisper`. Если `transcription_status = pending` — показываем плейсхолдер «обрабатывается». Если `failed` — «не удалось распознать». Если `done` с пустым текстом — «тишина или неразборчиво».

**Описание.** Такой же блок, лейбл `DESCRIPTION · GPT-4o`. Те же три состояния статуса.

**Кружок (video_note).** Показываем оба блока — и транскрипция, и описание. Они независимые, обновляются по разным событиям шины.

**Пересылка.** Над содержимым сообщения — тонкая вертикальная полоска слева и подпись «Переслано от @username» с датой. Данные — снимок на момент пересылки, не обновляются.

**Reply.** Аналогичная вертикальная полоска с серым цветом, подпись «собеседник:» или «вы:» и первые ~60 символов текста исходного сообщения. Клик — скролл к исходному сообщению.

**Редактирование.** Под временем — курсивная пометка «изменено HH:MM». Клик — поповер с `message_edits` по этому сообщению (старые версии текста с временем каждой).

**Soft delete.** Сообщение не скрывается, а показывается приглушённо (opacity 0.55) с красной плашкой сверху «УДАЛЕНО СОБЕСЕДНИКОМ · deleted_at HH:MM». Мы видели его, контекст разговора на нём опирается — поэтому показываем.

**Реакции.** Под сообщением — ряд пилюль со стандартными эмодзи и счётчиком. Наши реакции (`is_outgoing = true`) и реакции собеседника в одном ряду. Клик по реакции — показать кто и когда поставил (`created_at` / `removed_at`).

**Альбом (media_group_id).** Несколько сообщений с одинаковым `media_group_id` визуально слипаются в один пузырь с сеткой превью. Внутри — всё те же транскрипции/описания под каждым медиа.

## Бот и контакт — визуальные метки

| Условие | Визуальная метка |
|---|---|
| `dialogs.is_bot = true` | аватар окрашен в янтарный цвет |
| `dialogs.is_contact = true` | зелёная точка слева от имени |
| нет `username` | вместо @handle показываем «без username» курсивом, моноширинным |

## Поиск в списке диалогов

Простой текстовый поиск по полям `dialogs`:
- `first_name`, `last_name`
- `username`
- `phone`
- `contact_first_name`, `contact_last_name`

Без нечёткого поиска, без регулярок. `ILIKE %query%` — этого достаточно.

---

## Пагинация сообщений

Лента прокручивается вверх для загрузки старых сообщений — курсорная пагинация по `(date DESC, id DESC)`. Размер страницы — 50 сообщений.

При открытии диалога показываем последние 50. При скролле вверх — подгружаем следующие 50 до того которое уже есть.

---

## Что не делаем на этой странице в v1

- Не отправляем сообщения
- Не редактируем собственные сообщения
- Не удаляем сообщения
- Не ставим реакции
- Не загружаем медиа
- Не показываем "печатает..." и другие онлайн-состояния
- Не показываем счётчики непрочитанного — страница read-only, само понятие «прочитано» тут не применяется

Всё это возможно в будущем, но требует отдельных команд в API и враппере.

## Принципы которые мы зафиксировали

1. Страница только для просмотра — никаких изменений
2. Все данные — из таблиц модуля истории через API, не напрямую в БД
3. Исходящее / входящее — через `is_outgoing`, не через `sender_id`
4. Удалённые сообщения показываем приглушённо, не скрываем
5. Транскрипция и описание — независимые блоки, отображаются по своим статусам
6. Кружок имеет оба блока — транскрипция и описание
7. Пересылка — снимок на момент получения, не обновляется
8. Редактирование — пометка в сообщении, полная история правок в поповере
9. Бот и контакт имеют постоянные визуальные метки
10. Поиск простой — ILIKE по основным полям диалога
11. Пагинация сообщений — курсорная, по 50 штук
